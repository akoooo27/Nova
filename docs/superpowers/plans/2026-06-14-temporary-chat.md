# Temporary Chat Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let a chat be marked temporary at creation via an immutable `IsTemporary` flag on the `ChatThread` aggregate, persisted on the `chats` table, and set from a `?temporary-chat=true` query parameter on `POST /chats`.

**Architecture:** Add a plain immutable `bool` to the `ChatThread` aggregate (set in the `Create` factory, no mutator). Persist it as a non-null `is_temporary` column via an EF migration defaulting to `false`. Thread the flag from the create endpoint's query string → `CreateChatCommand` → handler → aggregate. The `SendMessage` path is untouched; it loads the thread and the flag rides along. Nothing reads or acts on the flag yet (retention, read-gating, memory suppression are out of scope).

**Tech Stack:** C# / .NET 10, EF Core (Npgsql, snake_case), FastEndpoints, Mediator, ErrorOr, xUnit.

**Spec:** `docs/superpowers/specs/2026-06-14-temporary-chat-design.md`
